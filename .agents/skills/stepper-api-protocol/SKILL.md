# Stepper API Protocol

When developing the Frontend (Phase 4) and the Backend Stepper API (Phase 3), you must strictly adhere to this communication protocol. 

## 1. Architectural Decision: REST over SSE
To keep the architecture robust and easy to debug, we will **NOT** use Server-Sent Events (SSE) for the multi-step wizard. Instead, we will use standard asynchronous REST POST requests for each step. The frontend will show a loading state while the backend agents process the step.

## 2. Universal Response Envelope
Every endpoint in the pipeline (`/api/pipeline/prepopulate`, `/api/pipeline/draft`, `/api/pipeline/format`, `/api/pipeline/validate`, `/api/pipeline/approve`) MUST return a JSON response matching this exact structure:

```json
{
  "status": "success" | "warning" | "error",
  "current_step": 1 | 2 | 3 | 4 | 5,
  "data": {}, // The payload specific to the step
  "message": "Human readable status or error message"
}
```

## 3. Step-Specific `data` Payloads

### Step 1: `/api/pipeline/prepopulate`
*   **Input:** `{"prompt": "Create Eldrin's wife"}`
*   **Success `data` Payload:**
    ```json
    {
      "detected_entities": ["Eldrin"],
      "prepopulated_schema": {
        "location": "Oakhaven",
        "faction": "Rangers",
        "relationships": {"spouse": "Eldrin"}
      }
    }
    ```

### Step 2: `/api/pipeline/draft`
*   **Input:** `{"prompt": "...", "schema_overrides": {...}}`
*   **Success `data` Payload:**
    ```json
    {
      "raw_draft": "Markdown text generated by lore_seeker..."
    }
    ```

### Step 3: `/api/pipeline/format`
*   **Input:** `{"raw_draft": "..."}`
*   **Success `data` Payload:**
    ```json
    {
      "yaml_frontmatter": {"name": "Lynda", "type": "character", ...},
      "markdown_body": "Prose content..."
    }
    ```
*   **Warning `data` Payload (if schema linter fails):**
    ```json
    {
      "missing_fields": ["age", "status"],
      "partial_yaml": {...}
    }
    ```

### Step 4: `/api/pipeline/validate`
*   **Input:** `{"yaml_frontmatter": {...}, "markdown_body": "..."}`
*   **Success `data` Payload:**
    ```json
    {
      "contradictions": [], // Array of strings if any exist
      "suggested_links": ["Oakhaven", "Eldrin"]
    }
    ```

### Step 5: `/api/pipeline/approve`
*   **Input:** Final frontmatter and body.
*   **Success `data` Payload:**
    ```json
    {
      "saved_path": "Knowledgebase/Characters/Lynda.md",
      "db_sync_status": "ok"
    }
    ```
